# Design Reference — Qlik Embed API Skill

Shared design system for pages generated by the Qlik Embed API skill. Uses CSS custom properties so all components adapt to theme changes by swapping only the `:root` block.

---

## Themes

Light is the default. Apply dark mode when the user requests it, or when embedding into a dark-themed application.

### Light Theme (Default)

```css
:root {
  --bg: #FAFAFA;
  --card-bg: #FFFFFF;
  --accent: #00873D;
  --text-primary: #404040;
  --text-muted: rgba(0, 0, 0, 0.55);
  --border: 1px solid rgba(0, 0, 0, 0.15);
  --card-shadow-hover: 0px 0px 0px 1px rgba(0, 0, 0, 0.05), 0px 2px 4px 0px rgba(0, 0, 0, 0.15);
  --header-border: 1px solid rgba(0, 0, 0, 0.06);
  --kpi-value-color: #1a1a1a;
  --chip-bg: color-mix(in srgb, var(--accent) 10%, transparent);
  --chip-border: color-mix(in srgb, var(--accent) 30%, transparent);
  --danger: #ef4444;
}
```

### Dark Theme

```css
:root {
  --bg: #0a0f1a;
  --card-bg: #111827;
  --accent: #15AA51;
  --text-primary: #f1f5f9;
  --text-muted: #94a3b8;
  --border: 1px solid rgba(255, 255, 255, 0.08);
  --card-shadow-hover: 0 8px 32px rgba(21, 170, 81, 0.15);
  --header-border: 1px solid rgba(255, 255, 255, 0.06);

  --chip-bg: rgba(21, 170, 81, 0.1);
```

> **Rule:** Use `var()` references everywhere in component styles. Never hardcode a color outside `:root`. If the user specifies brand colors, replace only the `:root` values.

## Typography & Readability Standards

Battle-tested sizes from real deployments. **Do not go smaller.**

```css
/* ── KPI Cards ──────────────────────────────── */
.kpi-value {
  font-family: 'Syne', sans-serif;
  font-size: 48px;        /* FIXED — never use clamp() — caps too small on wide screens */
  font-weight: 800;
  color: var(--kpi-value-color);
  line-height: 1;
  text-shadow: 0 0 30px color-mix(in srgb, var(--text-primary) 20%, transparent);
}

.kpi-label {
  font-size: 11px;
  letter-spacing: 0.14em;
  text-transform: uppercase;
  color: var(--text-muted);
  margin-bottom: 12px;
  font-weight: 500;
}

.kpi-sub {
  font-size: 11px;
  color: var(--text-muted);
  margin-top: 10px;
}

.kpi-card {
  padding: 32px 20px 28px;
  min-height: 160px;
}

/* ── Section Headings ───────────────────────── */
.section-heading h2 {
  font-size: 18px;        /* 14px is too small for dashboard headings */
  font-weight: 700;
  color: var(--text-primary);
}

/* ── Chart Card Titles ──────────────────────── */
.chart-card-title { font-size: 16px; color: var(--text-primary); }  /* not 13px */
.chart-card-desc  { font-size: 12px; line-height: 1.6; color: var(--text-muted); }  /* not 10px */
```

> ⚠️ **Lesson:** `clamp(22px, 2.8vw, 52px)` on a wide monitor locks KPI values at ~28px — unreadable. Always use **fixed 48px**. Always use **`var(--kpi-value-color)`** (not `var(--text-primary)`) for KPI numbers — primary text colors are often too dim for at-a-glance data.

## Embed Wrapper Sizing

Web components use shadow DOM — CSS attribute selectors don't reach inside. **Always wrap in a sized div:**

```css
.embed-wrap             { width: 100%; overflow: hidden; }
.embed-wrap.filter-size { height: 200px; }
.embed-wrap.kpi-size    { height: 160px; }
.embed-wrap.sankey-size { height: 500px; }
.embed-wrap.chart-size  { height: 400px; }
.embed-wrap.table-size  { height: 450px; }
```

| Object type | Recommended height | Notes |
|---|---|---|
| Filter panel | **200px minimum** | 110px clips values — unusable. Never go below 200px. |
| KPI | 160px | |
| Bar/line chart | 400px | |
| Sankey / complex | 500px | |
| Table | 450px | |

Include a subtle footer with:
- App ID (truncated)
- Tenant domain
- "Powered by Qlik" or "Embedded via Qlik Embed API"
- Last updated timestamp (static, set at generation time)
